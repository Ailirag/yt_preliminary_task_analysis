# Развёртывание ИИ-анализатора — пошаговая инструкция

Инструкция рассчитана на человека без опыта: делайте шаги подряд, копируя команды.
Весь путь занимает ~15–20 минут. Всё выполняется в окне **PowerShell**.

> В этой инструкции используется **подписка z.ai GLM Coding Plan** (модель GLM-5.2).
> Подписочный адрес API уже прописан в настройках; обычный (платный за токены) оставлен рядом
> закомментированным — как переключиться, см. **Приложение А**.

---

## Что понадобится (чек-лист)

- [x] Этот Windows-компьютер (проект уже лежит в папке `D:\Claude\yandex tracker task_bug analyzer`).
- [x] Доступ к трекеру — токены уже вписаны в систему (проверим на Шаге 4).
- [ ] **Подписка z.ai GLM Coding Plan** и **API-ключ** к ней (получим на Шаге 3).
- [ ] Папка с **выгрузкой конфигурации 1С УТ** в файлы (XML Конфигуратора) — укажем на Шаге 5.
- [ ] *(необязательно)* Небольшой баланс обычного тарифа z.ai — только если нужен разбор
      скриншотов (**Приложение Б**).

## Шаг 1. Открыть PowerShell в папке проекта

1. Откройте папку `D:\Claude\yandex tracker task_bug analyzer` в Проводнике.
2. Щёлкните в **адресную строку** Проводника (где написан путь), сотрите её, впишите `powershell`
   и нажмите **Enter**.
3. Откроется чёрное/синее окно — это и есть PowerShell, уже в нужной папке.

Проверьте, что вы в правильной папке — выполните:

```powershell
ls pyproject.toml
```

Если вывелась строка с `pyproject.toml` — всё верно. Если «не найдено» — вы не в той папке,
вернитесь к пункту 1.

---

## Шаг 2. Проверить вспомогательные программы

Проект использует две утилиты: **uv** (запуск Python) и **git**. Проверьте их:

```powershell
uv --version
git --version
```

Обе команды должны вывести номер версии. Если какая-то пишет «не является командой»:

- **uv** — установите командой: `powershell -c "irm https://astral.sh/uv/install.ps1 | iex"`,
  затем закройте и снова откройте PowerShell.
- **git** — скачайте и установите с https://git-scm.com/download/win (все галочки по умолчанию).

---

## Шаг 3. Получить API-ключ z.ai (подписка)

1. Если подписки ещё нет — оформите план на https://z.ai/subscribe (для пилота достаточно
   младшего плана **Lite**).
2. Откройте страницу ключей: **https://z.ai/manage-apikey/apikey-list**
3. Нажмите кнопку создания ключа (**Create API Key** / «Создать»).
4. **Скопируйте ключ целиком** и сохраните во временный текстовый файл — он понадобится на Шаге 4.
   Ключ показывается один раз; если потеряете — просто создадите новый.

> Один и тот же ключ работает и для подписки, и для обычного тарифа — различается только адрес,
> и он уже настроен правильно. Ничего дополнительно вписывать в адрес не нужно.

---

## Шаг 4. Вписать «переменные окружения» (самый важный шаг)

Переменные окружения — это способ передать программе секреты (токены, ключи), не записывая их
в файлы. Программа берёт их из системы автоматически.

**4.1. Проверьте токены трекера** (они уже должны быть вписаны ранее):

```powershell
if ($env:YATRACKER_TOKEN_GT) { "Токен трекера: ЕСТЬ" } else { "Токен трекера: НЕТ — обратитесь к тому, кто настраивал" }
if ($env:YATRACKER_ORGID_GT) { "ID организации: ЕСТЬ" } else { "ID организации: НЕТ — обратитесь к тому, кто настраивал" }
```

Должно вывести «ЕСТЬ» в обеих строках.

**4.2. Впишите ключ z.ai.** Замените `СЮДА-ВСТАВЬТЕ-КЛЮЧ` на ключ со Шага 3 (кавычки оставьте):

```powershell
[Environment]::SetEnvironmentVariable('ZAI_API_KEY', 'СЮДА-ВСТАВЬТЕ-КЛЮЧ', 'User')
```

**4.3. (Необязательно) Ключ Yandex** — только если планируете сравнивать с YandexGPT:

```powershell
[Environment]::SetEnvironmentVariable('YANDEX_API_KEY', 'СЮДА-КЛЮЧ-YANDEX', 'User')
[Environment]::SetEnvironmentVariable('YANDEX_FOLDER_ID', 'СЮДА-FOLDER-ID', 'User')
```

**4.4. ВАЖНО: закройте это окно PowerShell и откройте новое** (повторите Шаг 1).
Без перезапуска программа не увидит только что вписанный ключ. Это самая частая ошибка.

Проверьте в новом окне:

```powershell
if ($env:ZAI_API_KEY) { "Ключ z.ai: ЕСТЬ" } else { "Ключ z.ai: НЕТ — вернитесь к пункту 4.2 и перезапустите окно" }
```

## Шаг 5. Указать папку с выгрузкой конфигурации 1С

Анализатор читает исходный код конфигурации из **выгрузки в файлы** (XML Конфигуратора).

1. Откройте файл `config\analyzer.yaml` в Блокноте:

   ```powershell
   notepad config\analyzer.yaml
   ```

2. Найдите строку (примерно в середине файла):

   ```yaml
     dump_path: ""
   ```

3. Впишите между кавычек путь к папке выгрузки. **Слэши — прямые `/`**, например:

   ```yaml
     dump_path: "D:/1C/vygruzka-ut"
   ```

4. Сохраните файл (**Файл → Сохранить**) и закройте Блокнот.

> Нет готовой выгрузки? Её делает разработчик 1С через Конфигуратор
> (*Конфигурация → Выгрузить конфигурацию в файлы*). Без выгрузки анализатор всё равно запустится,
> но будет разбирать задачи **без доступа к коду** — качество ниже. Для пилота код желателен.

## Шаг 6. Установить проект (один раз)

```powershell
uv sync
```

Команда скачает зависимости (1–2 минуты). Должна закончиться без красных ошибок.

Проверьте, что модуль анализа кода 1С (`onec-vecgraph`) на месте:

```powershell
ls vendor\onec-vecgraph\pyproject.toml
```

Если файл есть — всё готово. Если «не найдено», выполните:

```powershell
git clone https://github.com/Ailirag/onec-vecgraph vendor/onec-vecgraph
```

## Шаг 7. Самопроверка

**7.1. Проверка окружения:**

```powershell
uv run analyzer preflight
```

Смотрите на столбец `[OK]` / `[FAIL]`. Что должно быть **OK**:

| Строка | Если FAIL |
|---|---|
| env YATRACKER_TOKEN_GT / ORGID | вернитесь к Шагу 4.1 |
| Tracker API /myself, Очередь ONE | нет интернета/VPN или отозван токен |
| Компонента «ИИ анализ» | уже создана (id=478); если нет — `uv run analyzer init-component` |
| Роль analyst (zai/glm-5.2) | не вписан ZAI_API_KEY — Шаг 4.2 + перезапуск окна |
| Выгрузка 1С (dump_path) | вернитесь к Шагу 5 |

**7.2. Проверка LLM (что ключ и модель реально работают):**

```powershell
uv run analyzer llm-test --provider zai
```

Для `glm-5.2` строки **чат**, **JSON**, **инструменты** должны быть `[OK]`.

Проверка разбора картинок (только если оформили обычный тариф — см. Приложение Б):

```powershell
uv run analyzer llm-test --model zai-payg/glm-4.6v
```

## Шаг 8. Первый прогон вхолостую (dry-run)

В этом режиме **в трекер ничего не записывается** — отчёты сохраняются в файлы, чтобы вы могли
их прочитать и оценить качество.

```powershell
uv run analyzer bugs --limit 3
```

По окончании откройте папку с отчётами:

```powershell
ls journal\dry-run
notepad journal\dry-run\ONE-XXXX.md
```

(подставьте номер задачи из списка, который вывела программа).

Прочитайте 2–3 отчёта. Если разбор адекватный — переходите к боевому режиму. Если нет —
покажите отчёты разработчику 1С и напишите, что поправить.

---

## Шаг 9. Боевой прогон (когда отчёты устраивают)

Здесь программа **создаёт подзадачи** с отчётом и **проставляет теги** на задачах.

1. Включите запись. Откройте конфиг:

   ```powershell
   notepad config\analyzer.yaml
   ```

   Найдите строку `mode: dry-run` и замените на:

   ```yaml
   mode: live
   ```

   Сохраните, закройте.

2. Запустите с флагом `--live`:

   ```powershell
   uv run analyzer bugs --limit 5 --live
   ```

Что произойдёт с каждой задачей: создаётся подзадача с компонентой «ИИ анализ» (в ней — отчёт),
родительской задаче добавляются теги `ИИ_анализ_проведен` и `ИИ-простая` / `ИИ-сложная`.

> **Два предохранителя.** Пока в файле стоит `mode: dry-run`, даже флаг `--live` ничего не запишет.
> Запись идёт только когда `mode: live` **и** указан `--live`.

**Полезные варианты запуска:**

```powershell
# только задачи, вручную помеченные тегом "к_анализу_ии":
uv run analyzer bugs --selection trigger-tag --live

# анализ готовых ФТ (тег "К анализу ИИ", ссылка на вики в поле «Ссылка на документацию»):
uv run analyzer ft --live

# одна конкретная задача:
uv run analyzer bugs --issue ONE-4740 --live
```

## Шаг 10 (необязательно). Автозапуск по расписанию

Чтобы анализатор запускался сам (например, дважды в день), зарегистрируйте задачу в Планировщике
Windows. Выполните **в PowerShell от имени администратора**:

```powershell
schtasks /Create /TN "TrackerAI-bugs" /SC DAILY /ST 07:30 /TR "\"D:\Claude\yandex tracker task_bug analyzer\scripts\run_scheduled.cmd\""
```

Задача будет запускать боевой прогон и писать лог в `journal\scheduler.log`.
Запускать от того же пользователя Windows, под которым вписаны ключи (Шаг 4).

## Приложение А. Переключение подписка ↔ обычный тариф (адрес API)

Адрес API задаётся в файле `config\providers.yaml`, провайдер `zai`. Сейчас активен подписочный:

```yaml
  zai:
    base_url: "https://api.z.ai/api/coding/paas/v4"   # ПОДПИСКА (Coding Plan) — активен
    # base_url: "https://api.z.ai/api/paas/v4"          # обычный тариф — запасной
```

Чтобы перейти на обычный тариф (оплата за токены): поставьте `#` перед подписочной строкой и
уберите `#` перед обычной. Сохраните файл. Ключ (ZAI_API_KEY) менять не нужно.

## Приложение Б. Разбор скриншотов (vision)

Подписка Coding Plan **не умеет разбирать картинки**. Скриншоты из задач анализирует отдельная
модель **GLM-4.6V** через **обычный тариф** z.ai (оплата за токены — нужен небольшой баланс).

- **Оставить включённым** (по умолчанию): в `config\providers.yaml` роль `vision: zai-payg/glm-4.6v`.
  Пополните баланс обычного тарифа в личном кабинете z.ai. Если баланса нет — прогон не упадёт,
  но в отчётах будет пометка «скриншоты не проанализированы».
- **Выключить совсем** (только текст задач, без картинок): в `config\providers.yaml` поставьте

  ```yaml
  roles:
    vision: none
  ```

## Приложение В. Частые проблемы

| Симптом | Решение |
|---|---|
| `Ключ z.ai: НЕТ` после Шага 4 | Не перезапустили окно PowerShell. Закройте и откройте заново. |
| В выводе «кракозябры» вместо русского | Выполните один раз в окне: `chcp 65001` и запустите команду снова. |
| `preflight`: Роль analyst FAIL | Не вписан или неверный ZAI_API_KEY (Шаг 4.2) либо нет подписки. |
| `llm-test` инструменты WARN/FAIL | Проверьте, что активен подписочный адрес (Приложение А) и подписка активна. |
| Анализ идёт «БЕЗ инструментов кода» | Не указан или неверный `dump_path` (Шаг 5), либо в папке нет `Configuration.xml`. |
| Ошибки `429` / `Retry` в логе | Трекер/LLM ограничивают частоту — программа сама повторяет запросы, просто подождите. |
| Хочу остановить прогон | Нажмите `Ctrl + C` в окне PowerShell. |

Журналы для разбора: `journal\runs.jsonl` (итоги по задачам), `journal\writes.jsonl` (что записано
в трекер), `journal\dry-run\*.md` (черновые отчёты).
